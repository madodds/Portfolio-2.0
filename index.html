Personal Projects

It Takes a Village

Unity 4.6 - Global Game Jam 2015

It Takes a Village (ITAV) is like Twitch Plays Pokemon meets Tamagotchi. ITAV is built to run on a server, streaming to twitch.tv, and accepting commands from the channel's chat group. The chat must attempt to keep the pet alive as long as possible by voting on what to do.

This was made for the Global Game Jam 2015, so we (a group of 3) only had a weekend to build it. I later refactored / de-bugged the code and put it on GitHub, where you can learn more and see the source code.

Game Idea Generator

AngularJS 1.4.4 / C# WinForms - 2015

Engine Website

Loading...

The Game Idea Generator randomly generates an idea for a video game project. This was originally written in C# windows forms, which you can find on GitHub.

The idea's attributes are retrieved from a JSON file on my GitHub account. The attributes chosen are based on randomly generated numbers that are unique to each other. These numbers get placed into the index of 5 arrays: Genres, Perks, Settings, Styles, and Engines.

Sphere Dash

Unity 3.7 - Graduation Project 2013

Sphere Dash is a side-scrolling platformer, where the environment is dynamically created. Your objective is to go very fast for as long as possible without falling off. There is no limit to how fast you can go. There are 3 different levels, and a tutorial level.

This game was originally made in 2012 as a project to learn Unity, but was expanded on as my graduation project. This game was judged as one of the top games of my graduating class.

About Me

For each "cell" I spawn a Rectangle of random width and length within some radius. Again the radius doesn't matter too much, but it should probably be proportionate to the number of cells. Instead of using uniformly distributed random numbers (the default Math.random generator in most languages), I'm using Park-Miller Normal Distribution[3] . This skews the size of the cells so that they are more likely to be of a small size (more smaller cells, less larger cells). The reason for this will be explained later! In addition to this I ensure that the ratio between the width and length of each cell is not too large, we don't want perfectly square rooms but neither do we want really skinny ones, but somewhere in between.
